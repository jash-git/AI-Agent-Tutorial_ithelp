Docker 介紹與安裝

李謦伊
7 min read
·
Jan 28, 2022

Docker 是一種基於 LXC (Linux Containers) 的輕量級虛擬化技術，利用 Linux kernel 的 Cgroup (Control Group) 和 Namespaces 功能建立獨立的容器 (Container)，能夠快速地進行開發、交付、部署、測試應用程式。透過 Docker 將當前的環境與應用程式封裝在一起，可以更方便地移植、部署到任何一個有安裝 Docker 的平台上。

Cgroup (Control Group) 用於控制、管理、分配系統資源的應用，如 CPU、Memory、IO。可參考 → 🔍

Namespace 用於區隔不同的環境。可參考 → 🔍

Docker 運行在 host machine 的核心上，也就是跟底層共用作業系統。不同於 VM (Virtual Machine，ex: VMware、VirtualBox) 需要藉由 Hypervisor (又稱為 virtual machine monitor，VMM) 來創建虛擬的環境，因此 Docker 的啟動、執行效率更快速，有以下幾點優勢:

  • 更快速的交付與部署
  • 高效能的虛擬化
  • 容易移植與擴展
  • 管理簡單化
  • 佔用記憶體空間較小

Docker 與 VM 的差別

VM 是在硬體層上進行虛擬化，因此需要在 VM 裡安裝作業系統，需佔用的硬體空間較大。此外，每一次安裝、開啟新的作業系統都要花費一些時間，也需要重新建立開發環境 (安裝需要的函式庫、應用程式等)，執行效率也較慢。

Docker 則是在作業系統層上進行虛擬化，共享 host machine 的作業系統，能夠減少硬體的使用空間。在每一次建立、啟動一個 container 只需要幾秒，並且執行效率接近原生系統，比起 VM 的效率更快速。每個 container 由 Docker Engine 所管理。

Docker 基本概念

Docker 的基本概念分為三部分：

  • 映像檔 (Image)

用來建立 Container 所需的應用程式環境，是一個唯獨的檔案，概念類似 VM 的 ISO 檔。

  • 容器 (Container)

Container 是由 Image 所建立執行的實例，可以被啟動、開始、停止、刪除。一個 Image 可以創建多個 Container，每個 Container 都是獨立運作、不互相影響。

  • 倉庫 (Registry)

用來管理 Repository 的場所，概念類似 Github。Repository 是指各種版本或標籤的 Images 集合，這些 Images 利用 tag 來區分。

Registry 分為公開倉庫 (Public) 以及私有倉庫 (Private)，目前最大的 Public Registry 為 Docker Hub；若需要用於自己測試或是公司內部使用的倉庫，則可以創建 Private Registry。

Docker 架構

Docker 採用 Client- Server 架構，Client 端為 Docker Client，通過 command line 或 Restful API 對 Docker daemon 發出請求；Server 端為 Docker daemon，是一個在後台持續運行的 process，負責管理 Docker 的物件，ex: Images、Containers、Networks、Volumes 等。

如下圖所示，Client 端發送指令傳給 Docker daemon，如: docker build、docker pull、docker run 等。接著Docker daemon 會依據該指令進行動作，如: 創建 image、產生 container、從 Registry 下載 image 至 local 端等。

安裝 Docker

依照官方的步驟進行安裝: https://docs.docker.com/engine/install/

我的作業系統是 Ubuntu 18.04，若要查看作業系統的版本可使用以下指令

$ cat /etc/lsb-release

安裝完 Docker 後使用以下指令確認安裝的版本

$ sudo docker version

若不使用 sudo (root 權限)，可以將使用者加入至 docker group

$ sudo groupadd docker$ sudo usermod -aG docker $USER$ newgrp docker# 執行 hello-world 確認 docker 安裝成功
$ docker run hello-world

Docker Image

接著來實際操作看看~ 首先從 Docker Hub 上取得需要的 image。這邊示範使用 mysql，使用 docker pull 將 image 拉至 local 端。預設是 latest 版本，也可以從 Tags 查看要下載哪種版本。

# latest version
$ docker pull mysql# 8.0.23 version
$ docker pull mysql:8.0.23

使用 docker images 查看目前已有的 image

Docker Container

有了 Docker Image 後就可以來運行容器~指令可以查看官方網站的 Start a mysql server instance

$ docker run --name mysql_v1 -e MYSQL_ROOT_PASSWORD=1234 -d mysql

執行 docker ps -a 查看容器是否運行成功。

接著就可以進入 Container 啦~ 若要退出 Container，可使用 exit 指令。

$ docker exec -it mysql_v1 bash

進入 Container後，執行以下指令連接 MySQL，密碼為上面設定的 MYSQL_ROOT_PASSWORD。若要退出 MySQL，可使用 exit 指令。

$ mysql -uroot -p

然後就可以在 MySQL Container 中執行指令，這邊示範查看 database

mysql> show databases;

刪除 image, container

在刪除 image 前，必須要先將其所有引用的 container 進行刪除。首先查詢 container id

$ docker ps -a

接著要注意：若需刪掉的 container 正在運行中，要先將其關閉

$ docker stop <container_id>

然後再進行刪除，執行 docker ps -a 可以看到該 container 被刪除了

$ docker rm <container_id>

若要一次刪除所有 container，可以使用以下指令

$ docker rm `docker ps -aq`

所有引用的 container 都刪除後，就可以刪掉 image 啦~ image_id 可使用 docker images 查看

$ docker rmi <image_id>
